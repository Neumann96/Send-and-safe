# SendBigFiles MVP

Telegram Mini App для анонимной передачи файлов с клиентским end-to-end
шифрованием. Backend никогда не получает ключ содержимого.

## Возможности

- файлы до 512 МБ;
- автоматическое удаление через 48 часов;
- без регистрации и без передачи Telegram ID;
- секретная ссылка с ключом после `#`;
- ссылка и отдельный 120-битный код доступа;
- AES-256-GCM и шифрование блоками по 4 МБ;
- SQLite для технических метаданных;
- локальный диск для зашифрованных блоков.

## Локальный запуск

Требования: Go 1.23+, Node.js 22+.

```powershell
cd web
npm install
npm run build
cd ..
go run ./cmd/server
```

Откройте `http://localhost:8080`.

Для разработки frontend:

```powershell
cd web
npm run dev
```

Vite проксирует `/api` на `http://localhost:8080`.

## Конфигурация

| Переменная | Значение по умолчанию |
| --- | --- |
| `ADDR` | `:8080` |
| `DATA_DIR` | `./data` |
| `WEB_DIR` | `./web/dist` |
| `MAX_FILE_BYTES` | `536870912` |

TTL намеренно не настраивается клиентом и всегда равен 48 часам.

## Модель безопасности

Ключ файла создаётся в браузере. Название, MIME-тип и содержимое файла
шифруются до отправки. Сервер видит размер, число блоков, IP-адрес и время
запросов, но не видит имя файла, код доступа или ключ расшифровки.

### Секретная ссылка

```text
https://host/d/TRANSFER_ID#key=FILE_KEY
```

Fragment после `#` не отправляется HTTP-серверу. Любой обладатель полной
ссылки может расшифровать файл.

### Ссылка и код

Приложение генерирует код из 24 символов алфавита Base32, то есть 120 бит
случайности. Через PBKDF2-SHA-256 с 310 000 итераций из кода выводится
AES-ключ, которым защищается ключ файла. На сервер отправляется только
зашифрованный ключ и параметры KDF.

Код и ссылку следует передавать разными каналами.

## Хранилище

SQLite содержит только:

- случайный ID передачи;
- SHA-256 хеш management/upload-токена;
- размер и количество блоков;
- статус и время истечения;
- публичный криптографический конверт.

Зашифрованные блоки находятся в `data/chunks`. Фоновая задача раз в минуту
удаляет истёкшие передачи.

## Telegram

Для публикации нужен HTTPS-домен и бот, созданный через BotFather. URL Mini
App должен указывать на корень приложения. Backend не использует
`initData`, поэтому Telegram-профиль пользователя не передаётся сервису.

Перед публичным запуском нужны внешний аудит протокола, rate limiting,
лимиты дисковой квоты, защита от злоупотреблений и объектное хранилище.
